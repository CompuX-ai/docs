# Contributing to CompuX

Thank you for your interest in contributing to CompuX! This repository contains the CompuX knowledge base, API examples, and documentation.

## Ways to Contribute

### Report Issues
Found a bug, typo, or inaccuracy? [Open an issue](https://github.com/CompuX-ai/compute-credit-financing/issues/new/choose) using one of our templates:
- **Bug Report** — something isn't working correctly
- **Content Request** — suggest new documentation or guides

### Join Discussions
Have a question or idea? Visit [Discussions](https://github.com/CompuX-ai/compute-credit-financing/discussions):
- **Q&A** — ask technical questions about the API or platform
- **Ideas** — suggest features or improvements
- **Show and Tell** — share what you've built with CompuX

### Improve Documentation
We welcome pull requests that:
- Fix typos or improve clarity
- Add code examples in new languages
- Translate documentation (see existing translations in README.*.md)
- Improve the [Wiki](https://github.com/CompuX-ai/compute-credit-financing/wiki)

### Add Code Examples
We'd love more integration examples:
- Different programming languages
- Framework integrations (LangChain, LlamaIndex, etc.)
- Use case demonstrations

## Pull Request Process

1. **Fork** the repository
2. **Create a branch** (`git checkout -b fix/typo-in-readme`)
3. **Make your changes** — keep PRs focused on a single change
4. **Test** — make sure examples run and links work
5. **Submit PR** — describe what you changed and why

### Content Guidelines
- Write in clear, concise English
- Use concrete examples and numbers where possible
- Keep paragraphs short (3–4 sentences max)
- Link to related pages within the knowledge base
- Include FAQ Q&A pairs when relevant

### Markdown Style
- Use `#` for page title, `##` for sections, `###` for subsections
- Use fenced code blocks with language identifiers
- Use tables for structured comparisons

## Code of Conduct

Be respectful, constructive, and inclusive. We're building something together.
