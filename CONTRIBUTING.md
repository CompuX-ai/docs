# Contributing to CompuX Docs

Thank you for your interest in contributing to CompuX documentation!

## How to Contribute

### Reporting Issues

- Found an error or outdated information? [Open an issue](https://github.com/CompuX-ai/docs/issues/new).
- Include the page URL and a description of the problem.

### Suggesting Improvements

- Have ideas for new content? Open an issue with the `enhancement` label.
- Want to improve an existing page? Submit a pull request.

### Submitting Changes

1. Fork this repository
2. Create a branch: `git checkout -b fix/your-fix-description`
3. Make your changes
4. Commit: `git commit -m "fix: description of change"`
5. Push: `git push origin fix/your-fix-description`
6. Open a Pull Request

### Content Guidelines

- Write in clear, concise English
- Use concrete examples and numbers where possible
- Keep paragraphs short (3-4 sentences max)
- Link to related pages within the knowledge base
- Include FAQ Q&A pairs when relevant

### Markdown Style

- Use `#` for page title, `##` for sections, `###` for subsections
- Use fenced code blocks with language identifiers
- Use tables for structured comparisons
- Front matter: include `title`, `description`, `date` fields
